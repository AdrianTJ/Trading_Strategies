# Project Review & Improvement Plan - May 15, 2026

## Project Status Overview
The StrategyTracker is functional but requires refinement in data management, calculation accuracy, and UI transparency regarding inflation. The core engine and data pipeline are robust, but the "last-mile" integration is incomplete.

## Identified Issues

### 1. Data Integrity (Backend)
- **Problem**: `run_simulations.py` appends results without clearing old ones, leading to duplicate entries in the dashboard.
- **Impact**: UI shows redundant strategies; database grows unnecessarily.

### 2. Calculation Errors (Frontend)
- **Problem**: `ReturnsMatrix.tsx` calculates monthly returns in isolation (last day vs. first day of the month).
- **Impact**: Misses the overnight/weekend return between months, resulting in inaccurate performance data.

### 3. Missing Roadmap Features (UI)
- **Problem**: "Real Returns" (inflation-adjusted) are calculated in the backend but not displayed in the frontend.
- **Impact**: Fails to meet the core value proposition of showing the impact of inflation on investment strategies.

### 4. Configuration Rigidity
- **Problem**: Asset and strategy lists are hardcoded in `App.tsx`.
- **Impact**: Scaling to new assets requires code changes rather than just configuration.

## Improvement Plan

### Phase 1: Data Management & Backend
- [x] **Fix Duplication**: Update `save_simulation` in `scripts/run_simulations.py` to delete previous runs for the same asset/strategy combination before saving new ones.
- [x] **Data Cleanup**: Provide a utility to reset simulation tables.

### Phase 2: Frontend Accuracy & Metrics
- [x] **Correct Returns Matrix**: Refactor `ReturnsMatrix.tsx` to use the closing value of the previous month as the baseline for the current month.
- [x] **Add Real Returns**: Update `StatsGrid.tsx` and `ComparisonTable.tsx` to display Inflation-Adjusted (Real) CAGR and Total Return.
- [x] **Total Invested Stat**: Add a "Total Capital Invested" metric to clearly distinguish between DCA and Lump Sum cash requirements.

### Phase 3: Developer Experience & Polish
- [x] **Asset Config**: Move asset/strategy metadata to a configuration file or fetch from API.
- [x] **Test Integration**: Fix `PYTHONPATH` issues and add a `npm test` script that handles both frontend and backend tests.
